# Controllable Chain-of-Thought Compression: Accuracy, Length, and Energy

This repository contains the code to reproduce the experiments on **controllable
Chain-of-Thought (CoT) compression** and its effect on the trade-off between
reasoning length, task accuracy, and inference energy. The method (selectively
skipping redundant CoT tokens at an adjustable compression ratio) is applied to
several open-weight instruction models and evaluated on reasoning/QA benchmarks and
an execution-based multilingual code benchmark.

The study covers **four base models** — Qwen2.5-3B-Instruct, Qwen2.5-7B-Instruct,
Qwen2.5-14B-Instruct, and Llama-3.1-8B-Instruct — across the benchmarks **GSM8K,
MATH-500, BoolQ, PIQA** (reasoning/QA) and **McEval** (multilingual code).

## Repository layout

```
.
├── README.md
├── common/      # shared engine + tooling for the reasoning/QA benchmarks
├── gsm8k/       # GSM8K       : data + configs + run.sh
├── math/        # MATH-500    : data + configs + run.sh
├── boolq/       # BoolQ       : data + configs + prepare.py + run.sh
├── piqa/        # PIQA        : data + configs + prepare.py + run.sh
├── mceval/      # McEval (execution-based multilingual code) — self-contained package
└── plots/       # figure generation from aggregated result summaries
```

The four reasoning/QA benchmarks share one `--benchmark`-parameterized engine, so
each benchmark folder is a thin, self-contained working directory (its dataset,
configs, and a `run.sh`) and the reusable code lives once in [`common/`](common).
`mceval/` is an independent pipeline with its own package and README.

## Method and external components

The approach builds on **TokenSkip** (controllable CoT compression):
- TokenSkip — https://github.com/hemingkx/TokenSkip (arXiv:2502.12067)

It also uses these external, publicly available components:
- **LLMLingua-2** — CoT compressor used to build the compressed training data.
- **LLaMA-Factory** — LoRA supervised fine-tuning framework.
- **McEval** — execution-based multilingual code benchmark and Docker harness
  (`multilingualnlp/mceval`), used by the `mceval/` arm.

Please cite the corresponding upstream works when using this code.

## Hardware used

All training and inference were run on a single server:

| Component | Specification |
|---|---|
| CPU | Intel Xeon Gold 6326 |
| RAM | 256 GB |
| GPU | 2 × NVIDIA RTX A6000 (49 GB VRAM each) |

The 3B/7B and 8B models run on a single GPU; the 14B model uses tensor parallelism
across the two GPUs for inference. Energy measurements combine on-GPU power
(`nvidia-smi`) with rack-level active power read over SNMP from a PDU; the PDU host
in the scripts is a placeholder (`192.0.2.1`) and must be set to your own host.

## Software environments

Two conda environments are used (the names are conventions, not requirements):

| Environment | Purpose |
|---|---|
| `tokenskip_env` | vLLM inference, TokenSkip, LLMLingua-2 compression, parsing, analysis |
| `llamafactory_env` | LoRA supervised fine-tuning (LLaMA-Factory) |

The reasoning/QA arm's pinned dependencies are in
[`common/requirements.txt`](common/requirements.txt) (notably `torch==2.5.1`,
`vllm==0.6.4.post1`, `transformers==4.47.0`, `peft==0.14.0`). The `mceval/` arm is a
CPU-light Python package whose heavy, server-only dependencies (vLLM, LLaMA-Factory,
LLMLingua-2) are installed into the conda environments; see its own README. The
`mceval/` evaluation also requires the **McEval Docker image**, pulled by its
`sha256` digest (never a floating tag) so the matrix of language runtimes is fixed.

## Reproducing the experiments

> Base models and benchmark datasets are downloaded from their public sources
> (Hugging Face). Pin the exact model commit hashes for an exact reproduction. Bulk
> artifacts (model outputs, compressed corpora, adapter/merged weights, evaluation
> dumps) are **not** included; they are regenerated by the pipeline.

### A. Reasoning / QA benchmarks (`gsm8k/`, `math/`, `boolq/`, `piqa/`)

Per benchmark and model, the pipeline (shared code in [`common/`](common)):

1. **Prepare data** — `boolq/prepare.py`, `piqa/prepare.py` download/format the raw
   data (GSM8K and MATH-500 data are already provided under their `datasets/`).
2. **Compress the CoT** — `common/LLMLingua.py` / `common/LLMLingua_iterato.py`
   build compressed CoT training data across the compression-ratio grid.
3. **Build SFT data** — `common/get_llamafactory_input_*.py` convert it into
   LLaMA-Factory training files.
4. **LoRA fine-tune** with LLaMA-Factory (example configs in
   `common/configs/examples/train_lora/`).
5. **Merge the adapter** into the base model with `common/merge.py`
   (`common/scripts/merge_qwen.sh` / `merge_llama.sh`).
6. **Evaluate** — a single run from the benchmark folder:
   ```bash
   cd gsm8k && bash run.sh        # edit model + compression ratio inside run.sh
   ```
   For the full ratio × token sweep with energy logging, use the drivers in
   [`common/scripts/`](common/scripts) (see that folder's README).

### B. Multilingual code (`mceval/`)

Orchestrated end-to-end by `mceval/scripts/run_pipeline.py` (train-data generation →
compression → LoRA SFT → energy-instrumented evaluation sweep), stopping at two
human review gates. The authoritative recipe is
[`mceval/docs/PIPELINE_RUNBOOK.md`](mceval/docs/PIPELINE_RUNBOOK.md); design and
decisions are in [`mceval/docs/PROJECT_ROADMAP.md`](mceval/docs/PROJECT_ROADMAP.md),
and the workflow in [`mceval/docs/WORKFLOW.md`](mceval/docs/WORKFLOW.md). Copy the
`mceval/configs/*.example.yaml` templates and fill in machine-specific paths and the
pinned run metadata (model commits, tool versions, McEval Docker digest).

### C. Figures (`plots/`)

The figures regenerate from the aggregated result summaries (no re-running needed):
```bash
cd plots && pip install -r requirements.txt && python plot_curves_normalized.py
```
See [`plots/README.md`](plots/README.md).

## Configuration and paths

No identifying or machine-specific absolute paths are committed. Paths come from
config files or environment variables. For `mceval/`, copy the provided
`*.example.yaml` templates and edit them. Set the PDU host / SNMP settings
(placeholder `192.0.2.1`) to your own infrastructure, or disable energy logging if
not needed.
